### FORMATO DE SAÍDA (OBRIGATÓRIO)

Retorne **APENAS um array JSON válido com exatamente {n_posts} itens**.

Cada item deve seguir **EXATAMENTE** esta estrutura:

```json
{
  "title": "...",
  "summary": "...",
  "source_url": "...",
  "source_type": "...",
  "arxiv_id": "...",
  "suggested_angle": "...",
  "illustration_hint": "...",
  "raw_sources": [
    {
      "title": "...",
      "url": "...",
      "content": "...",
      "score": 0.0,
      "source": "..."
    }
  ]
}
```

---

### DESCRIÇÃO DOS CAMPOS

* **title**
  Título específico e técnico. Deve refletir o insight principal (evite genéricos).

* **summary**
  Resumo técnico (2–4 frases) explicando:

  * o que foi proposto/construído
  * por que isso importa na prática
  * onde isso se encaixa (ex: agentes, RAG, inference, etc.)

* **source_url**
  Link principal (priorize arXiv ou repositório oficial)

* **source_type**
  Um dos valores:

  * `"arxiv"`
  * `"github"`
  * `"blog"`
  * `"release"`

* **arxiv_id**
  ID do paper (ex: `"2604.03201"`) ou `null` se não for arXiv

* **suggested_angle**
  Ângulo técnico e opinativo para o post.
  Deve ser:

  * específico
  * prático
  * orientado a sistemas reais
    Exemplo: impacto em produção, trade-offs, limitações, custo vs performance

* **illustration_hint**
  Um dos valores:

  * `"paper_figure"` → diagramas do paper
  * `"repo_image"` → imagens do repositório
  * `"code"` → snippet técnico relevante
  * `"stock_photo"` → apenas se não houver alternativa técnica
  * `"none"`

* **raw_sources**
  Lista das fontes originais usadas para compor o tópico.
  Inclua 1 a 5 itens relevantes extraídos dos resultados brutos.
  Cada item deve conter:

  * `title`: título da fonte
  * `url`: link
  * `content`: trecho relevante
  * `score`: score original (não altere)
  * `source`: origem (ex: `"tavily"`, `"arxiv"`)

---

### REGRAS FINAIS

* Retorne apenas o JSON (sem explicações)
* Não invente dados que não estejam nos resultados
* Prefira profundidade técnica > popularidade
* O conteúdo deve ser útil para quem constrói sistemas reais
* O `"suggested_angle"` deve guiar diretamente o writer
